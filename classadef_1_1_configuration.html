Detailed Description

Configuration provides interfaces to access configuration data.

There are three kinds of the configuration data:

  • object : It consists of members.
    • member: name-value pair.
      name is string and value is value accessed through the name.
  • array : It consists of elements.
    • elements: value accessed through the ordered index.
  • value : It can be a null, true, false, string, number (including int, unsigned int, and double), object or array.

Configuration use the Builder pattern to create configuration data. It generates the Concrete Builder when it loads a configuration file, then it use the builder to create configuration data.
See the Builder pattern

See also
ConfigurationBuilder, ConfigurationData.
